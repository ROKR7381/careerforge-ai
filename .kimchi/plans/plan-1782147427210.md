# Plan: Smart Job Discovery — Phase 1

## Goal

Make `/jobs` auto-populate a ranked, personalised job feed (sourced via Adzuna, with match scores from the user's saved resume) — replacing the current paste-and-match UI.

## Constraints

- **No direct Naukri/LinkedIn scraping.** Source via Adzuna aggregator (legally indexes both).
- **Reuse existing infra** — `/api/match-score`, Prisma `Resume` model, existing visual primitives.
- **No new npm deps** — hand-rolled Adzuna fetch wrapper.
- **Truthful source attribution** — Adzuna's `source` field only, never fabricated.
- **Phase 1 only.** All deferred items (Jooble, affiliate, email alerts, salary insights, Google for Jobs, ATS one-click apply, multi-resume, saved-job expiry, webhooks, alert preferences) are explicitly out of scope and will be re-discussed with the user after Phase 1 ships and is verified working.

## Chunks

**Scope:**

- `prisma migrate dev --name add_saved_jobs` (run locally; committed as migration file)

### Chunk 2 — Aggregator library: Adzuna client + query builder + deep-links + cache
**Scope:**
- `src/lib/job-aggregator/adzuna.ts` — `search(params): Promise<Job[]>`. Reads `ADZUNA_APP_ID`/`ADZUNA_APP_KEY` from env. Normalises Adzuna's response into our `Job[]` shape (handles 401, 429, 500, network error).
- `src/lib/job-aggregator/query-builder.ts` — `resumeToQuery(resume): JobSearchParams`. Pure function. Handles all 8 edge cases (empty resume, fresher, senior, career-switcher, freelancer, no-location, no-skills, multi-experience).
- `src/lib/job-aggregator/deep-links.ts` — `buildDeepLinks(params): { naukri: string, linkedin: string }`. URL-encodes properly.
- `src/lib/job-aggregator/cache.ts` — `get/set/invalidate` with 24h TTL. Implementation: try Redis if `REDIS_URL` is set, else in-memory `Map` with timestamp-based expiry. Sufficient for MVP.

**Depends On:** Chunk 1
**Accept When:**
- Unit tests pass for all 4 files (see Verification Strategy)
- Adzuna client returns normalised `Job[]` against the live API using a test query
- Cache round-trip works and TTL expires correctly
**Open Questions:** none

- Use existing `/api/match-score` for scoring — do NOT duplicate the algorithm.

**Depends On:** Chunk 1, Chunk 2

## Decision Log

| Decision | Choice | Rationale | Rejected |
|---|---|---|---|
| Job source | Adzuna aggregator | Legally indexes Naukri/LinkedIn/Indeed; ToS-clean; free tier covers MVP | Direct scraping (ToS violation, IP bans, legal risk); Indeed publisher API (manual approval needed) |
| Brand trust for Naukri/LinkedIn | Deep-link buttons with pre-filled queries | Zero scraping cost, user keeps brand recognition | None — only viable path |
| Match scoring | Reuse existing `/api/match-score` | Already works; no duplicate algorithm | Build new (waste) |
| Cache layer | In-memory Map with TTL; Redis if `REDIS_URL` set | Zero infra for MVP; Redis-ready | Redis-only (forces infra on day 1) |
| Saved-job storage | New `SavedJob` Prisma model | Clean separation from `Resume` | JSON column on `User` (hard to query, no relations) |
| Resume data source | Latest saved resume by `updatedAt desc` | Matches user's most recent intent | Multiple-resume picker (YAGNI for Phase 1) |
| Scope | Phase 1 only | User explicitly chose 1-week MVP ship | Phase 1+2+3 (too big, no validation yet) |
| Post-Phase-1 next step | **Stop and ask the user** | User explicitly instructed | Auto-proceed to deferred items |

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| Adzuna India coverage is thin for niche roles | Medium | Dedupe + filter logic; expose "Edit query" so user can broaden; Phase 2 adds Jooble as backup |
| Match-score API is slow for 25 jobs serial | Medium | Concurrency cap of 5; cache layer shields repeat visits |
| Adzuna rate-limits free tier (250 calls/mo) | Low–Medium for MVP | 24h cache per user; surface usage in admin; Phase 2: paid tier + Jooble |
| Saved jobs reference dead URLs (post removed) | High over time | Document as known limitation; Phase 2 adds expiry/re-validation |
| User has multiple resumes and wants to pick | Medium | Document as deferred; user can manually switch in `/builder` |
| Adzuna schema changes | Low | Pin to current schema in `adzuna.ts`; add a unit test for response shape |
| Adzuna returns jobs with `source: null` (rare) | Low | Fall back to "Adzuna" badge; never fabricate a source |
| Python backend down → match-score fails for all jobs | Medium | Cache layer keeps last good result; UI shows "scoring unavailable" badge |
| Resume has no skills/title (very new user) | Low | Fall back in `query-builder.ts`; UI prompts to complete resume |
